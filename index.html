How to turn on/off crosshairs/hitmarker

Used files:
scripts\hudlayout.res
resource\clientscheme.res

The default file structure looks like:

    Cross //1.1 .............................. crosshair name and number you looking for
    {
    "controlName" "CExLabel"
    "fieldName" "Cross"
    "visible" "0" ............................ "1" to enable visibility, "0" to disable
    "enabled" "1"
    "zpos" "2" ............................... Z axis (use it when multiple xhairs enabled)
    "xpos" "c-25" ............................ X axis from center of screen
    "ypos" "c-26" ............................ Y axis from center of screen
    "wide" "50" .............................. use it for more accurate X axis positioning if needed
    "tall" "49" .............................. use it for more accurate Y axis positioning if needed
    "font" "CrosshairsOutline16" ............. used font parameter (look for avalable fonts in clientscheme.res)
    "labelText" "a"
    "textAlignment" "center"
    "fgcolor" "255 255 255 255" .............. RGBA (Red, Green, Blue, Alpha) color (0 min, 255 max)
    }
For example, let's do yellow crosshair, enabled only in minmode. So we're need to add/change some parameters:
    "visible" "0" ............................ now it will be disabled when cl_hud_minmode "0"
    "visible_minmode" "1" .................... you can use "_minmode" with any parameter you want
    "fgcolor" "255 255 0 255" ................ because red+green=yellow in RGB color model
To enable the custom crosshair(s), open open scripts\hudlayout.res, then CTRL+F for needed number from table and change "visible" and/or "visible_minmode" parameter values to "1". Then you can calibrate xhair positions with "xpos", "ypos" and "wide", "tall" too.

To disable the hitmarker, open scripts\hudlayout.res, then CTRL+F for "3.1" and change "visible" and "visible_minmode" parameter values to "0".

How to change damage position and visibility

Used files:
resource\ui\huddamageaccount.res
resource\ui\huddamageaccount_4x3.res
resource\ui\huddamageaccount_16x10.res
resource\ui\huddamageaccount_m0re.res
resource\ui\huddamageaccount_yellow_4x3.res
resource\ui\huddamageaccount_yellow_16x9.res
resource\ui\huddamageaccount_yellow_16x10.res

The default file structure looks like:

    "CDamageAccountPanel" .................... damage/healing numbers near the enemy
    {
    "fieldName" "CDamageAccountPanel"
    "text_x" "0"
    "text_y" "0"
    "delta_item_end_y" "0"
    "PositiveColor" "255 255 50 255" ......... damage colour (RGBA)
    "NegativeColor" "50 255 50 255" .......... healing colour (RGBA)
    "delta_lifetime" "3.0" ................... how long last damage numbers stay visible on screen
    "delta_item_font" "DefaultOutline" ....... font used for damage numbers near the enemy
    "delta_item_font_big" "DefaultOutline" ... font used for much damage done
    }

    "CDamageAccountValue" .................... damage/healing numbers near the health
    {
    "ControlName" "CExLabel"
    "fieldName" "CDamageAccountValue"
    "fgcolor" "255 255 50 255" ............... RGBA color
    "font" "HudFontBiggerBold" ............... used font parameter
    "xpos" "c-302" ........................... X axis from center of screen
    "ypos" "r110" ............................ Y axis from right
    "xpos_minmode" "c-316" ................... X axis from center of screen (minmode)
    "ypos_minmode" "r75" ..................... Y axis from right (minmode)
    "zpos" "2" ............................... Z axis
    "wide" "100" ............................. box wide (change when damage value looks "cropped"
    "tall" "26" .............................. box height (change when damage value looks "cropped"
    "visible" "1" ............................ visibility parameter
    "enabled" "1"
    "labelText" "%metal%"
    "textAlignment" "left"
    }

To enable alternative damage layouts, just rename huddamageaccount_yellow.res or huddamageaccount_m0re.res to huddamageaccount.res (placed in resource\ui\ folder). "4x3", "16x9" and "16x10" means screen aspect ratio used to calibrate positioning.

How to change scoreboard layout

Used files:
resource\ui\scoreboard.res
resource\ui\scoreboard_6v6.res
resource\ui\scoreboard_640x480.res

All you need to know:

By default, 6v6 table enabled only when cl_hud_minmode "1". Scoreboard_6v6.res contains 6v6 table for both minmode parameters, so it works when cl_hud_minmode "0" and "1". Scoreboard_640x480.res is a special version of scoreboard.res, optimised for using on low screen resolution. If you don't need 6v6 table anyway and wants to see default one, so just delete scoreboard.res file placed in resource\ui\ folder.

How to change Medic HUD layout

Used files:
resource\ui\hudmediccharge.res
resource\ui\hudmediccharge_alt_4x3.res
resource\ui\hudmediccharge_alt_16x9.res
resource\ui\hudmediccharge_alt_16x10.res
resource\ui\hudmediccharge_alt_combined_16x9.res
scripts\hudanimations_tf.txt
scripts\hudanimations_tf_alt.txt
scripts\hudanimations_tf_alt_combined.txt

So, what's in the box?

By default, the small ubercharge value placed near the crosshair and default uberbar/value saved at their regular positions too. In hudmediccharge_alt.res you can find bigger ubercharge value replaces the default one. Also, here is combined layout too, with bigger default ubercharge/value replacement and small ubercharge value under the crosshair. Hudanimations files used for better supporting new layouts, adding more conspicuous colors when ubercharge is ready to drop.

Not recommended to use with Vaccinator.
